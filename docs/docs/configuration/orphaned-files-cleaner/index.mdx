# Orphaned Files Cleaner

The Orphaned Files Cleaner scans configured directories for files and folders that are no longer tracked by any active torrent in your download client, and moves them to a dedicated orphaned directory. Optionally, it can purge entries from that directory after a configurable number of days.

<Important>
Supported for all download clients: qBittorrent, Transmission, Deluge, rTorrent, and uTorrent. The scan runs as part of the Download Cleaner job — it uses the same schedule. Configure these settings in the **Download Cleaner** page.
</Important>

<ConfigSection
  title="Exclude Patterns"
>

Glob patterns for file or folder names that should never be considered orphaned, even if no active torrent claims them. Matching is case-insensitive.

**Examples:**
```
*.nfo
*.txt
.DS_Store
Thumbs.db
```

<Note>
Patterns are matched against the entry name only (not the full path). Use `*` as a wildcard for any characters.
</Note>

</ConfigSection>

<ConfigSection
  title="Min File Age"
>

Minimum age in minutes a file or folder must have before it can be considered orphaned. This protects files that are actively being downloaded or just finished.

Set to `0` to disable age filtering (default).

**Example:** Set to `60` to ignore files modified less than an hour ago.

</ConfigSection>

<ConfigSection
  title="Purge Orphaned After"
>

Number of days after which entries in the Orphaned Directory are permanently deleted. Leave empty to keep orphaned files indefinitely.

**Example:** Set to `30` to automatically purge orphaned files older than 30 days.

<Warning>
Files deleted by this option are permanently removed from disk. Make sure you have reviewed the contents of your Orphaned Directory before enabling this setting.
</Warning>

</ConfigSection>

</div>

<div className={styles.section}>

<SectionTitle>Per Download Client</SectionTitle>

<p className={styles.sectionDescription}>
  Each download client has its own scan directories and orphaned directory. Configure these under the <strong>Orphaned Files</strong> accordion inside the Download Cleaner per-client section.
</p>

<ConfigSection
  title="Enabled (per client)"
>

Enable orphaned files scanning for this specific download client. The client must also be enabled globally in Download Client settings.

</ConfigSection>

<ConfigSection
  title="Scan Directories"
>

A list of absolute paths to scan for orphaned files on this download client. Each top-level entry inside these directories is checked against your active torrents' save paths. If no active torrent claims an entry, it is considered orphaned.

**Examples:**
```
/data/downloads/completed
/data/downloads/cross-seed
```

<Important>
Each path must be accessible by Cleanuparr. If running in Docker, make sure to mount the directories accordingly.
</Important>

<Important>
Scan directories must not overlap with those of other download clients. Cleanuparr enforces this at save time to prevent cross-client false positives.
</Important>

</ConfigSection>

<ConfigSection
  title="Orphaned Directory"
>

The directory where orphaned files and folders are moved for this download client. Leave empty to log orphaned entries without moving them.

**Example:**
```
/data/downloads/orphaned
```

<Note>
The orphaned directory itself is never scanned for orphans.
</Note>

</ConfigSection>

<ConfigSection
  title="Download Directory Source and Download Directory Target"
>

The path prefix as reported by this download client (`Source`) and the corresponding path accessible to Cleanuparr (`Target`). Used to translate torrent save paths so Cleanuparr can correctly determine which files are claimed.

Leave both fields empty if no path translation is needed.

**Docker example:**

```yaml
services:
  qbittorrent:
    volumes:
      - /host/data/downloads:/downloads   # qBittorrent sees /downloads

  cleanuparr:
    volumes:
      - /host/data/downloads:/data/downloads  # Cleanuparr sees /data/downloads
```

In this case:
- **Download Directory Source**: `/downloads`
- **Download Directory Target**: `/data/downloads`

qBittorrent reports save paths starting with `/downloads`. Cleanuparr translates them to `/data/downloads` before comparing against the scan directories.

</ConfigSection>
