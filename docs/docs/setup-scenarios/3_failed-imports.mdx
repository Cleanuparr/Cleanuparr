# Using Cleanuparr just for failed imports (torrent and usenet)

1. Enable the `Queue Cleaner`.
2. Set the `Failed Imports Max Strikes` to a desired value.
3. Optionally set failed import message patterns to ignore potentially valid items from being processed.

<Important>
    When no download client is configured, no other action involving a download client would work (e.g. content blocking, removing stalled downloads or excluding private trackers).
</Important>
